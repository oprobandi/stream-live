# ADR-009: V1.3 Frontend Improvements

**Status:** Accepted  
**Date:** 2026-03-23  
**Version:** V1.3.0

---

## Context

V1.2 established a stable, deployable static platform. The V1.x backlog (tracked in `docs/TODO.md`) accumulated several 🟡 Important items that improve usability without requiring a backend. V1.3 addresses all of them in a single release.

---

## Decisions & Rationale

### 1. Upload persistence via IndexedDB (`idb.js`)

**Problem:** Uploaded videos were stored as `URL.createObjectURL()` blob references in memory. A page reload invalidated all uploads.

**Solution:** New `SL.idb` module wraps the browser IndexedDB API. On upload finalisation, the raw `File` object (not just a blob URL) is stored keyed by video id. On `store.init()`, all persisted records are loaded and fresh blob URLs are generated. This means uploads survive page reloads and browser restarts as long as the browser retains IndexedDB data.

**Trade-off:** IndexedDB is not available in `file://` origin in some browsers. `store.init()` wraps the idb call in try/catch — if unavailable, the app degrades gracefully to in-memory only behaviour with a console warning.

### 2. Keyboard focus trap in modals (`SL.a11y`)

**Problem:** Opening any modal allowed Tab to move focus outside the modal overlay, violating WCAG 2.1 SC 2.1.2 (No Keyboard Trap in reverse — focus should be *contained* in a dialog).

**Solution:** `SL.a11y.trapFocus(modal)` is called on modal open in `auth.js`, `premium.js`, `upload.js`, and `player.js`. It adds a keydown listener that cycles focus within the modal's focusable elements. `SL.a11y.releaseFocus()` removes the listener and restores focus to the previously focused element on close.

### 3. Scroll position memory

**Problem:** Closing the player modal jumped the page to the top.

**Solution:** `SL.player` now stores `window.scrollY` on open and calls `window.scrollTo({ top: this._scrollY, behavior: 'instant' })` on close.

### 4. Video poster attribute

**Problem:** The `<video>` element showed a black rectangle while buffering.

**Solution:** `player.open()` sets `videoEl.poster = video.thumb` before loading the source. Card thumbnails are also used as poster via `onload`/`onerror` hooks that remove a `skeleton-loading` class.

### 5. Skeleton loaders

**Problem:** On slower connections the video grid flashed unstyled/empty before images loaded.

**Solution:** `SL.views.renderSkeletons()` renders animated shimmer placeholder cards immediately. `store.init()` is `async` — skeletons show while IndexedDB loads, then `renderGrid()` replaces them with real cards. Individual card thumbnails also use a `skeleton-loading` class removed by the image's `onload` event.

### 6. Share button + deep linking (`?v=<id>`)

**Problem:** No way to share a specific video.

**Solution:** `SL.player.open()` calls `history.replaceState()` to append `?v=<id>` to the URL without a page reload. On close, the param is removed. `SL.player.share()` uses the Web Share API if available (mobile), falls back to Clipboard API, then falls back to a toast. On app init, `?v=` is read and the video is auto-opened via `handleVideoClick`.

### 7. Mobile search

**Problem:** The search bar was hidden on screens < 640px with no alternative.

**Solution:** A search icon button (`#nav-search-toggle`) appears only on mobile (CSS `display:none` by default, `display:flex` at ≤ 640px). Clicking it toggles a `.mobile-open` class on `.nav-search-wrap`, which slides the search bar in below the navbar as an absolutely positioned overlay. Clicking outside closes it.

### 8. Dark / Light mode toggle

**Problem:** The design system used hardcoded dark variables with no toggle.

**Solution:** A `[data-theme="light"]` block overrides all CSS custom properties. `SL.app.toggleTheme()` flips `document.documentElement.dataset.theme` and persists the choice to `localStorage`. The icon in `#theme-toggle` updates to sun/moon accordingly.
